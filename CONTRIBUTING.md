#  Contributing Guidelines

Thank you for contributing to the Smart Staff Attendance & Payroll Management System.

This project was developed collaboratively by Software Engineering students using GitHub workflow practices.

---

#  Contribution Rules

##  Before Contributing

- Pull latest project updates
- Create meaningful commits
- Test code before pushing

---

#  Coding Standards

- Use clear variable names
- Maintain modular C++ structure
- Keep functions focused and reusable
- Follow object-oriented programming principles

---

#  Database Guidelines

- Maintain relational integrity
- Use proper SQL formatting
- Avoid redundant data structures

---

#  Git Workflow

## Clone Repository

```bash
git clone https://github.com/HUNDEX-create/SmartPayrollSystem.git
```

---

## Create Branch

```bash
git checkout -b feature-name
```

---

## Commit Changes

```bash
git add .
git commit -m "Describe your changes"
```

---

## Push Changes

```bash
git push origin feature-name
```
